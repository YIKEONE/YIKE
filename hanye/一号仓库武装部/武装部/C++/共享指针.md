在C++中，共享指针是一种智能指针，用于管理动态分配的内存，并且能够跟踪有多少个指针指向同一块内存区域。最常见的共享指针是 `std::shared_ptr`，它是C++标准库提供的一种智能指针。

共享指针的工作原理是使用引用计数。当一个 `std::shared_ptr` 对象被创建时，它会分配一个相关联的引用计数。每当有一个新的 `std::shared_ptr` 指向相同的内存时，引用计数就会增加。当 `std::shared_ptr` 被销毁或不再指向相应的内存时，引用计数就会减少。当引用计数减少到0时，表示没有任何 `std::shared_ptr` 指向该内存，这时内存会被释放。

下面是一个简单的示例说明了如何使用 `std::shared_ptr`：
```c++
#include <iostream>
#include <memory>

class MyClass {
public:
    MyClass() {
        std::cout << "MyClass constructed." << std::endl;
    }
    ~MyClass() {
        std::cout << "MyClass destructed." << std::endl;
    }
    void DoSomething() {
        std::cout << "Doing something." << std::endl;
    }
};

int main() {
    std::shared_ptr<MyClass> ptr1(new MyClass); // 创建第一个 shared_ptr，引用计数为1
    {
        std::shared_ptr<MyClass> ptr2 = ptr1; // 创建第二个 shared_ptr，引用计数增加为2
        std::shared_ptr<MyClass> ptr3(ptr2); // 创建第三个 shared_ptr，引用计数增加为3
        ptr2->DoSomething(); // 通过任意一个 shared_ptr 访问对象
    } // ptr2 和 ptr3 超出作用域，引用计数减少为1
    ptr1->DoSomething(); // 通过剩下的 shared_ptr 访问对象
    // 当 ptr1 超出作用域，引用计数减少为0，对象被销毁
    return 0;
}

```
在这个示例中，`ptr1`、`ptr2`和`ptr3`都指向同一个 `MyClass` 对象。当最后一个指向该对象的 `std::shared_ptr` 超出作用域时，`MyClass` 对象会被销毁，因为引用计数减少到0。这样可以确保在不再需要对象时释放相应的内存，从而避免内存泄漏。

在C++中，除了`std::shared_ptr`，还有另一种智能指针叫做`std::weak_ptr`。`std::weak_ptr` 是一种弱引用智能指针，它允许你观察到被 `std::shared_ptr` 管理的对象，但不会增加该对象的引用计数。

弱引用指针通常用于解决由共享指针可能导致的循环引用问题。循环引用发生在两个或多个对象相互持有对方的共享指针时。这会导致对象永远无法被销毁，因为它们的引用计数永远不会降为0。

下面是一个结合了`std::shared_ptr`和`std::weak_ptr`的示例：
```c++
#include <iostream>
#include <memory>

class B; // 前向声明

class A {
public:
    std::shared_ptr<B> b_ptr; // A 持有一个指向 B 的 shared_ptr
    A() {
        std::cout << "A constructed." << std::endl;
    }
    ~A() {
        std::cout << "A destructed." << std::endl;
    }
};

class B {
public:
    std::weak_ptr<A> a_weak_ptr; // B 持有一个指向 A 的 weak_ptr
    B() {
        std::cout << "B constructed." << std::endl;
    }
    ~B() {
        std::cout << "B destructed." << std::endl;
    }
};

int main() {
    std::shared_ptr<A> a_ptr(new A); // 创建一个 shared_ptr 指向 A
    std::shared_ptr<B> b_ptr(new B); // 创建一个 shared_ptr 指向 B
    a_ptr->b_ptr = b_ptr; // A 持有对 B 的共享指针
    b_ptr->a_weak_ptr = a_ptr; // B 持有对 A 的弱引用指针

    // 输出 A 和 B 对象是否还存在
    std::cout << "a_ptr.use_count() = " << a_ptr.use_count() << std::endl;
    std::cout << "b_ptr.use_count() = " << b_ptr.use_count() << std::endl;

    // 注意：即使 B 持有 A 的弱引用，但不会增加 A 的引用计数
    // 因此，当 a_ptr 超出作用域时，A 对象会被销毁，但 B 对象仍然存在
    return 0;
}

```
在这个示例中，`A` 类持有一个指向 `B` 对象的`std::shared_ptr`，而 `B` 类则持有一个指向 `A` 对象的`std::weak_ptr`。这样设计的好处是，即使 `B` 类持有对 `A` 对象的引用，但并不影响 `A` 对象的生命周期。当 `A` 对象的最后一个 `shared_ptr` 被销毁后，`A` 对象会被正确地销毁，而不会因为 `B` 对象的存在而导致循环引用和内存泄漏。
